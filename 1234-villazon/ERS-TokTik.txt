# Documento de Requisitos de Software TokTik

## 1. Introducción

El software será una réplica de TikTok donde se pague por una suscripción para tener acceso al contenido, que en lugar de ser de entretenimiento, será informativo/de noticias.

### 1.1. Alcance del Producto

Se espera que el software sea un software de consumo masivo, y por ende tenga bastante alcance y necesite mayor soporte para no colapsar.

### 1.2. Valor del Producto

El software va a permitir a los usuarios acceder a información verificada y confiable, evitando posibles fake news o información falsa que circula en el resto del Internet.

### 1.3. Público Objetivo

El público objetivo serán usuarios de todo el mundo interesados en información científica, histórica o de actualidad.

### 1.4. Uso Previsto

Se espera que el software sea usado creando una cuenta, pagando una suscripción mensual, y donde se pueda ya sea ver la información publicada en forma de videos cortos (de menos de 1 minuto como en TikTok), o subir su propia información. Los usuarios además podrán denunciar contenido falso para mantener la plataforma limpia.

### 1.5. Descripción General

El software funcionaria de la misma forma que TikTok, solamente que mostrará información verificada en formato de videos de 1 minuto o menos. Además, el software tendrá métodos de pago para suscribirse y tener acceso a las noticias. 

## 2. Requisitos

### 2.1. Requisitos Funcionales

- Los usuarios deben poder registrarse y crear una cuenta.

- Se debe ofrecer un sistema de suscripción con métodos de pago digitales.

- Los usuarios pueden ver videos informativos de hasta 1 minuto de duración.

- Se debe permitir a los usuarios subir videos con información verificada.

- Implementación de un sistema de moderación para reportar y eliminar contenido falso.

- Algoritmo de recomendación basado en los intereses del usuario.

- Funcionalidad de comentarios y reacciones en los videos.

- Posibilidad de compartir contenido en otras plataformas.

### 2.2. Requisitos de Diseño

- La interfaz debe ser intuitiva y similar a la de TikTok para facilitar la experiencia del usuario.

- El sistema debe permitir la verificación de cuentas para creadores de contenido confiables.

- Implementación de un reproductor de video optimizado para carga rápida.

- El sistema debe soportar múltiples idiomas.

- Diseño adaptable a dispositivos móviles y tablets.

### 2.3. Requisitos No Funcionales

- El sistema debe ser escalable para soportar una gran cantidad de usuarios simultáneamente.

- La plataforma debe garantizar la seguridad de los datos personales y de pago de los usuarios.

- La velocidad de carga de los videos debe ser óptima, con un tiempo de espera mínimo.

- El sistema debe estar disponible 24/7.

- Se debe cumplir con normativas de privacidad y protección de datos.
